# CHANGELOG

## v1.1 (2026-03-27)

### 变更
- [修复] 测试指令不再包含 expected，实现盲测原则
- [新增] inputs 目录结构，记录测试输入（prompt.md）
- [新增] tester 输出规范：result.md（必需）+ test-record.md（可选）
- [新增] 评估指令要素模板
- [修改] 测试目录位置不再限制，默认 test_space/
- [修改] 测试流程增加 inputs 创建步骤
- [修改] 阶段三和阶段四的流程描述

### 关键决策
- **盲测原则**：测试 SubAgent 执行时不知道预期效果，避免"暴露答案"
- **输入输出分离**：expected 只存在于 eval_metadata.json，评估阶段才使用
- **职责分离**：测试 SubAgent 只输出结果，评估 SubAgent 负责判断

## v1.0 (2026-03-24)
初始版本，完成技能快速测试方法。

### 变更
- [新增] 四阶段测试流程：制定方案 → 用户确认 → 执行测试 → 评估结果
- [新增] 场景选择与准备（5步法 + 发散-收敛思维）
- [新增] 两种指令系统（测试指令 + 评估指令）
- [新增] eval_metadata.json 规范
- [新增] 可选基线对比功能
- [新增] 测试失败处理和报告生成

### 关联决策
- 与 skill-creator 的定位区分：快速验证 vs 量化测试
- 采用"快速、隔离、真实、可复现"的核心原则
