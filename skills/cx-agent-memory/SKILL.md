# agent-memory

**让 Agent 拥有持续性记忆的能力。**

Agent 的上下文窗口有限，一旦会话结束，之前所有的讨论、决策、约定都会丢失。通过将关键信息持久化到记忆库，未来的 Agent（即使是全新实例）可以快速恢复上下文，避免重复询问、重复犯错。

**如果不使用这个技能会怎样：**
- 下次会话需要重新询问用户已解释过的事情
- 同一项目的工作风格不一致，反复变化
- 用户需要反复纠正相同的偏好设置

**这个技能的本质：** 不是"可选的增强功能"，而是维持长周期项目一致性的基础设施。

## 约定

### 核心约束

确保下次阅读时能准确理解主旨。

### 存储结构

- `index.md` 是核心索引文件
- 其他文件的组织方式由 Agent 自行决定

### 记忆库位置

Agent 自行决定记忆库存放位置，创建后应在 `index.md` 中说明。建议在 index.md 的"项目概述"或顶部注释中标注记忆库路径。

### 启动协议

启动协议是 AGENTS.md 中的标准流程，用于会话开始时恢复上下文：

```
1. 检查记忆库 index.md 是否存在
2. 若存在，读取 index.md 获取项目持久化记忆
3. 根据任务类型，按需加载记忆库中的资料
4. 运行后续操作
```

**注意**：启动协议（读取记忆库）≠ 编辑前加载技能。读取 index.md 不需要加载本技能，编辑记忆库文件时才需要触发。

## 行为指引

### 查询

查询比记录更频繁，优先考虑查询效率：

1. 从 `index.md` 开始查找
2. 使用 `grep` 和 `glob` 搜索历史内容
3. 根据项目特点选择合适的查询方式

### 记录

**触发时机**：
- 完成重要决策后
- 用户明确要求记录
- 会话结束前

**执行步骤**：
1. 首次记录时，读取 `references/writing-guide.md`
2. 后续按需参考
3. 若 `index.md` 不存在，创建并初始化（写入强制协议声明，见 writing-guide.md）

**内容判断**：
- 记录：重要决策、解决方案、关键发现、上下文依赖
- 不记录：临时状态、可重建信息、冗余细节

### 整理

**触发时机**：
- 用户明确要求整理
- Agent 自行判断需要整理（见下方判断依据）

**执行步骤**：
1. 读取 `references/maintenance.md`
2. 按其中的**诊断-决策-执行-验证**框架进行自主判断和执行

**判断依据**（Agent 自行决定是否触发整理，以及何时触发）：
- 信息过于分散，难以定位关键内容
- `index.md` 摘要区域失去快速定位作用
- 同一主题内容分散在多个文件
- 内容存在错误、过时信息或与实际工作出入较大
- 存在大量无效的文件链接
- 内容有重复或冗余
- 文件数量过多或目录分布混乱

## AGENTS.md 集成

本技能与 AGENTS.md 启动协议的关系：

### 启动协议（AGENTS.md）

用于**会话开始时**恢复上下文，属于被动读取：
```markdown
## 启动协议
1. 检查记忆库 `index.md` 是否存在
2. 若存在，读取 index.md 获取项目持久化记忆
3. 根据任务类型，按需加载记忆库中的资料
4. 运行后续操作
```

### 编辑记忆库（本技能）

用于**编辑记忆库时**主动管理，需要触发本技能：
- 触发条件见 description 中的触发场景
- 执行流程见本文件的"行为指引"

### 何时帮助用户编写 AGENTS.md

- 用户明确要求时
- 首次创建记忆库时（可建议用户添加到 AGENTS.md）
- 不主动修改已有 AGENTS.md

## 异常处理

### 记忆库不存在

**场景**：首次使用或记忆库被删除

**处理**：
1. 主动告知用户即将创建记忆库
2. 在约定位置创建 `index.md`
3. 写入技能声明和初始化信息

### index.md 损坏/格式错误

**场景**：文件损坏、无法解析

**处理**：
1. 尝试读取文件，检测到损坏
2. 备份原文件（重命名为 `index.md.bak`）
3. 重建空的 `index.md`，写入技能声明
4. 从备份文件中提取未损坏的内容

### 引用文件不存在

**场景**：index.md 中引用的链接指向已删除文件

**处理**：
1. 标记为失效链接（搜索确认文件是否真的不存在）
2. 清理 index.md 中的失效链接
3. 如文件实际存在但路径错误，修正路径
4. 不主动搜索记忆库中是否有替代文件

### 权限问题

**场景**：无法创建/修改文件

**处理**：
1. 检测到权限不足
2. 告知用户权限问题，建议检查目录权限
3. 不尝试绕过权限或创建临时文件替代

### 并发冲突（多 Agent 同时操作）

**场景**：多个 Agent 实例同时修改记忆库

**处理**：
- 本技能不处理并发冲突，由上层调用方保证串行执行
- 如检测到明显的写入冲突，保留时间戳更新的版本

## 决策框架

### "Agent 自行决定" 的判断维度

当技能中提及"Agent 自行决定"时，使用以下维度判断：

| 决策点 | 判断维度 |
|--------|---------|
| **记忆库位置** | 用户偏好 / 当前工作目录结构 / 是否已有约定位置 |
| **文件组织方式** | 项目规模 / 内容类型复杂度 / 历史习惯 |
| **是否需要记录** | 信息是否影响后续决策 / 是否需要跨会话传递 |
| **是否需要整理** | 查找效率是否下降 / 是否有明显的结构混乱 |
| **内容粒度** | 主题独立性 / 未来可能变更频率 / 与现有文件的重叠度 |

**原则**：记录是为了未来的自己理解——如果犹豫，就问"3个月后看这条记录，我能否快速理解"。
