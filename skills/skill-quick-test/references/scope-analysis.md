# 范围分析

> 📍 本文件是 skill-quick-test skill 的参考文件。主入口和决策树请见 SKILL.md。

---

## 目标

从被测技能的 SKILL.md 提取信息，为场景设计提供输入。本文件对应测试设计方法论的**步骤 1：识别输入空间**。

---

## 步骤

### 1. 读取入口

读取被测技能的 `SKILL.md`，提取以下四类信息：

#### 功能域

技能声称能处理什么？逐项列出。

**来源**：description、能力边界声明、图表/功能选择指南。

**输出格式**：
```
功能域:
  - 绘制流程图
  - 绘制 ER 图
  - 跨平台兼容性查询
  ...
```
每行对应一项独立声称。

#### 路径集

Agent 可以怎样进入和使用这个技能？

| 类型 | 识别方法 | 触发复杂测试？ |
|------|---------|:--:|
| **正门** | SKILL.md 主入口（按推荐路径进入） | 是 |
| **侧门** | 参考文件直接进入（跳过 SKILL.md） | 否 |
| **回访** | 已知文件名直接打开 | 否 |
| **引用跳转** | 文件间链接跳转 | 否 |

每种类型至少列出一个具体起点（文件路径）。

#### 决策点

技能中有 Agent 需要做选择的地方吗？

- 有显式决策树 → 提取每个分支条件
- 有"如果...则..."类指引 → 提取每个条件判断
- 没有 → 标记"无决策点"

#### 能力边界

技能声明的范围和限制。

**来源**：能力边界声明、版本号、平台支持表、"支持 X 种"类数字。

**输出格式**：
```
边界声明:
  - 声称支持 11 种图表类型（SKILL.md:L19）
  - 支持 GitHub / GitLab / VS Code / Obsidian / Notion / Typora（SKILL.md:L37-45）
  - 不支持"特定平台的安装配置教程"（SKILL.md:L153）
```

这是边界测试（步骤 3）的输入。

### 2. 委派需求检测

扫描被测技能的全部文件中是否引用了子代理委派行为（`delegate_task` 或委派指令）。

- 如有 → 标记 ⚠️，列出涉及的功能点或路径
- 如无 → 无标记

涉及委派的路径在测试中由测试子代理自行模拟。

---

## 输出

完成分析后，输出整合报告：

```
被测技能: [名称] v[版本]

功能域 (N 项):
  - ...

路径集:
  正门: [具体起点]
  侧门: [具体起点]
  回访: [文件路径]
  引用跳转: [A → B]

决策点: [有/无] — [简述或分支列表]

能力边界:
  - ...（来源: 文件:行号）

委派需求: [有/无] — [涉及的功能点]
```

这份报告是 `scenario-design.md` 的输入。
