# 任务流水线型框架（Task Pipeline Framework）

## 适用场景

**特征**：
- 输入明确、输出明确
- 处理步骤可以标准化
- 边界清晰，不需要太多判断
- 可重复执行

**典型智能体**：
- 数据处理助手
- 文档格式化助手
- 信息提取助手
- 翻译助手

## 与行为定义型的关键差异

| 维度 | 任务流水线型 | 行为定义型 |
|------|-------------|-----------|
| **决策需求** | 低（按步骤执行） | 高（需要判断） |
| **方法描述** | 步骤清单（第一步、第二步...） | 思维模式（如何思考、如何选择） |
| **灵活性** | 低（固定流程） | 高（根据情况调整） |
| **适用场景** | 边界清晰的执行任务 | 需要专业判断的分析任务 |

## 设计要点

1. **步骤要可执行**：每一步都能明确判断是否完成
2. **输入输出要清晰**：每一步的输入输出定义清楚
3. **要有错误处理**：考虑各种异常情况
4. **要有质量标准**：如何判断最终结果好坏

## 步骤设计原则

**每个步骤必须包含**：
- 明确输入
- 具体操作
- 明确输出
- 完成标准

**示例**：
```markdown
**步骤2：数据清洗**
- 输入：原始数据（含噪声）
- 操作：
  1. 去除空值
  2. 去除重复
  3. 格式标准化
- 输出：清洗后的数据
- 完成标准：无空值、无重复、格式统一
```

## 使用提示

1. **适合重复任务**：如果每次执行步骤都一样，用这个框架
2. **不适合复杂判断**：如果需要大量决策，用行为定义型
3. **强调可预测性**：输出应该是可预期的
4. **错误处理很重要**：流水线中任何一步失败都需要处理
