# Research Workflow

📍 `web-deep-research` · 线索追踪与交叉比对方法论 · [← 回 SKILL.md](../SKILL.md)

> Phase 0–1 明确话题并建立线索起点。Phase 2 纯收集——追踪、阅读、总结、连接，不做判断，全程记录 URL。Phase 3 交叉比对——根据资料性质（事实 / 知识 / 意见 / 数据 / 经验）采用不同的评估方式。发现新问题 → 回到 Phase 2 补充研究。Phase 4 交付——整理研究报告，资料不被结论覆盖。

---

## Phase 0: 明确话题（不联网）

**目标**：在搜索之前，确保你知道用户到底在问什么。不联网——仅基于对话上下文和你的判断。

**产出**：记录以下三项（不需要格式化，但必须可被后续阶段直接引用）：
- 确认后的实体名称（Phase 1 用来验证指称，之后锁定不再偏移）
- 目标清单，逐条列出（Phase 1 用来映射线索覆盖；Phase 2 定期锚定；Phase 4 按此组织结论）
- 范围边界（Phase 2 定期回看是否偏离；Phase 4 结论范围锚定）

范围可在研究中途调整，但调整需要显式标注。

### 指称确认

用户用的名称是正确的吗？

- 已知实体 → 直接确认
- 听说的名字 / 内部代号 / 不完整名称 → 评估：你能自信推断正确名称吗？
  - 能 → 记录推断结果，Phase 1 搜索中验证
  - 不能 → **直接向用户确认**
- 无具体实体名称（类别查询：如"有什么好看的 X""最近 X 领域的新动态"）→ 确认类别边界：时间范围、质量标准、类型边界。这些边界的澄清本身也是研究的一部分——不联网时先做初步界定并标注不确定性，Phase 1 搜索中用真实数据修正

### 消歧

一个名字指向多个实体时，判断用户指的是哪一个。

- 上下文足以区分 → 自行判断
- 上下文不足，歧义无法调和 → **直接向用户确认**

### 意图拆解

用户一句话中可能包含多个需要独立研究的维度：

- **显式目标**：用户明确要求回答的问题
- **隐式目标**：用户语句中隐含的研究需求（如"帮我们设计 Agent"隐含了"Agent 设计方法论有哪些"）

每个目标作为一个独立维度纳入 Phase 1 的线索映射。

完成指称确认和意图拆解后，检查：当前话题是否匹配 Content Index 中的任何领域文档？回顾 SKILL.md 的 Content Index 表格，确认所有匹配的领域文档都已加载，若未加载则现在加载。

### 确定范围

本次研究包含什么、不包含什么。边界可以在研究过程中调整，但调整需要显式标注。

## Phase 1: 建立线索起点（联网）

**目标**：开始搜索，用真实数据验证 Phase 0 的判断，建立第一批线索。

### 操作

1. **验证指称**——用 Phase 0 确认的名称首次搜索。发现偏差（如"有戏AI 已更名为 XX"）→ 立即更新名称，之后锁定，不再偏移
2. **初始搜索**——用最终确认的名称搜索话题，浏览 5-10 个结果
3. **识别线索**——从搜索结果中提取：
   - 术语：不熟悉的概念、专业名词、缩写
   - 人物：被反复提及的名字、机构、作品
   - 事件：时间节点、转折点、争议
   - 来源：被引用的原始报告、访谈、论文——记录它们，这是 Phase 2 的种子
4. **映射维度**——对照 Phase 0 拆解出的所有目标，逐条映射：每个目标有哪些已知线索？哪些目标的线索稀疏甚至空白？
5. **语言选择**——根据话题选择搜索语言：
   - 创意/媒体话题：英文搜索通常线索更丰富（中文搜索可能被无关内容淹没）
   - 本土产业/政策话题：以本地语言为主
   - 技术话题：中英文并用
   - 如果选了单一语言但发现线索稀疏，换一种语言试试

### 完成检查

- 指称是否已验证或修正？
- Phase 0 拆解出的每个目标都有线索覆盖吗？无覆盖的目标——为什么搜不到？（语言问题？搜索词问题？还是这个维度本身信息稀缺？）

## Phase 2: 追踪线索链

**目标**：追踪线索，阅读，总结，发现新线索。**此阶段不做真假判断、不做置信度评估**——只收集和连接。判断留给 Phase 3。

### 操作

1. **取出下一条线索**——优先追踪来自高权威来源的线索，优先追踪能填补稀疏维度的线索
2. **阅读原始内容**——不只看摘要。不只看转载。直接阅读这个来源本身的完整内容
3. **总结**——用你自己的话概括这个来源说了什么。总结不是判断——不要说"这是对的"或"这不可信"，只复述它的说法。概括时保留具体信息（数字、日期、原文引述），不要压缩成笼统描述
4. **追踪连接**——这个来源：
   - 引用了什么？（追溯被引用方）
   - 被谁引用？（查找引用它的后续来源——这是发现新视角的路径）
   - 提到了哪些新术语、新人名、新事件？
5. **记录来源**——每条信息标注出处，**必须包含 URL 地址**（或文档路径、出版物编号等可追溯标识）。同时分配一个唯一的来源编号 `[S1]`, `[S2]` ...——按收集顺序递增即可。预判资料类型（事实 / 数据 / 知识 / 意见 / 经验）——可粗略，Phase 3 复核修正。这个编号贯穿后续所有阶段，在 Phase 3 的发现和 Phase 4 的报告中直接引用。来源丢失 = 这条信息在 Phase 3 无法被评估
6. **追加到线索清单**——新发现的线索加入待追踪清单，标注它属于 Phase 0 的哪个维度
7. **定期锚定**——回看 Phase 0 的判断：当前追踪是否仍在回答原始问题？是否偏离了范围？

### 组织节奏

不同话题的线索追踪节奏不同，选择适合的即可：

- **链式**：线索 A → A 引用的来源 B → B 引用的来源 C。深度优先，适合有清晰引用链的话题
- **广度优先**：先收集话题下所有主要来源，再逐个深读。适合框架、方法论、知识体系类话题
- **矩阵填充**：有多个实体和多个维度时，可以按维度横扫或按实体竖填。适合竞品分析、多方案对比

### 停止条件

研究自然结束时通常意味着：

- 新线索不再带来新信息，只是在重述已知内容
- Phase 0 拆解出的每个目标都有信息覆盖——自问：如果此刻就进入 Phase 3 交叉比对，每个目标至少有一条线索可以拿出来比对吗？如果某个目标连一条可用的线索都没有，那还没到停止的时候
- 剩余未追踪线索的边缘价值很低

不需要逐条打勾。但如果"感觉做完了"，回看一下——有没有一个维度线索特别稀疏但你下意识跳过了？

## Phase 3: 交叉比对

**目标**：对 Phase 2 收集到的全部资料进行系统比对。根据资料的**性质**采用不同的评估方式——事实用置信度，知识用合理性/系统性，意见找争议。交叉比对中发现的新问题→回到 Phase 2 追踪。

### 第一步：分类

在比对之前，先判断你手头的每条关键信息属于什么类型：

- **事实**：关于现实世界中发生过/存在着的状态的声明。"X 公司成立于 2023 年""产品定价 ¥29/月"。**有真假值**——它要么符合现实，要么不符合
- **数据**：看起来像事实，但本质是"某人按某方法在某时间点测出的数字"。"市场规模 30 亿美元""用户增长 150%"。可靠与否取决于**测量方法**，而非数字本身是否能被直接验证
- **知识 / 框架**：系统化的认识结构。"三幕结构""Egri 的角色骨骼理论""精益创业方法论"。**没有真假值**——它要么内部自洽、有解释力、适用于你的场景，要么没有。不存在"三幕结构是对的还是错的"这个问题
- **意见 / 立场**：带有评价的声明，混了事实基底和主观判断。"A 是市场领导者""这个方向没有前途"。需要拆开：事实基底部分按事实处理，评价部分标注为立场
- **经验**：情境化的实践知识。"我们团队发现 X 做法在 Y 场景下导致 Z 问题"。有用性取决于**情境描述是否足够具体**、是否可迁移到你的场景

### 第二步：按类型分别处理

以下各类比对中，每产生一条发现即分配一个唯一编号 `F1`, `F2` ...，并标注它依据的 [S 编号]。这些编号在 Phase 4 中直接带入报告，不需要重新编号。

#### 事实 → 置信度评估

**目标**：区分可靠事实和错误信息。

- 多条高权威来源独立指向同一事实 → 高置信度
- 来源权威性分层：一手（原始报告、访谈原文、官方数据）> 权威二手（专业媒体、学术论文、行业报告）> 一般来源（个人博客、论坛）。一条学术论文 + 两条个人博客一致，不等于"三方确认"
- 只有一条来源覆盖 → 标注"单来源，未经交叉验证"，不赋置信度
- 高权威来源之间说法矛盾 → 这是有价值的发现，如下处理：
  - 如果矛盾涉及核心问题 → 转化为新线索，回到 Phase 2 追踪"为什么存在这个矛盾"
  - 如果矛盾不涉及核心 → 标注并记录各方说法，不强行裁定
- 低权威来源与高权威来源矛盾 → 标注，采信高权威

#### 数据 → 方法论追溯

**目标**：判断数字的来源是否可靠，而不是判断数字本身是否"正确"。

- 追溯：谁测的？用什么方法？样本是什么？统计口径是什么？（如"市场规模"是否含了某个子品类？"用户数"是注册用户还是月活？）
- 如果数据的测量方法和定义边界不可追溯 → 标注为"来源不明数据"，不应作为关键结论的支柱
- 不同来源对同一指标给出了不同数据 → 分析差异原因（口径不同？方法论不同？时间点不同？），而非简单判断"哪个对"

#### 知识 / 框架 → 系统性组合

**目标**：理解框架之间的关系，组合成一个系统化的知识图景。

- 框架之间是**互补**关系？还是**替代**关系？还是**层级**关系（一个是大框架，另一个是其子组件）？
- 多个框架覆盖同一问题但来自不同流派 → 对比它们的核心假设和适用范围。这是分析产出，不是需要解决的矛盾
- 框架内部是否自洽？是否与已知的高置信度事实冲突？（框架本身不能是"错的"，但它可以与事实矛盾——此时需要标注"某框架的某推论与某事实不一致"）
- 是否适用于你的场景？——这是对框架的**适配性判断**，不是对框架本身的评估

#### 意见 / 立场 → 归纳与争议识别

**目标**：理解各方立场，识别实质性争议。

- 归纳：在这个话题上存在哪些不同的立场？每个立场的持有者是谁？他们的相关性和发言资格是什么？（不是权威——一个 CEO 对行业的看法和一个学者的看法都有相关性，但维度不同）
- 识别实质性争议：不是"有人说好有人说坏"这种浅层分散——而是**基于不同前提或不同价值观得出的系统化分歧**
- 如果发现了实质性争议 → 标注争议的本质（前提冲突？价值冲突？信息不对称？），可能转化为新线索回到 Phase 2 探索

#### 经验 → 可迁移性评估

**目标**：判断他人的经验能否作为你的参考。

- 情境描述是否具体到你能判断它和你场景的相似度？
- 经验中提到的因果链是否清晰？（"我们做了 X，观察到了 Y，分析后认为原因是 Z"）还是只有结论没有过程？
- 如果可迁移 → 标注适用条件和限制
- 如果不可评估（太笼统）→ 标注，不作为行动依据

### 第三步：发现新问题

交叉比对中可能出现以下情况：

- 事实矛盾无法裁定 → 需要更多信息来打破僵局
- 某个维度的所有线索都来自低权威来源 → 需要寻找高权威补充
- 某个 Phase 0 目标完全缺乏覆盖 → 需要针对性搜索
- 意见争议的根源值得深入理解 → 需要进一步研究争议各方的论据

这些新问题 → 回到 Phase 2，作为新的线索追踪（S 编号和 F 编号从当前最大值继续递增）。

## Phase 4: 整理交付

**目标**：将 Phase 2 收集的资料和 Phase 3 的比对结果整理为研究报告。

**核心原则**：报告中的每一条结论都可以被读者逆向追踪——结论 ← 发现 ← 依据（资料摘要）← 来源（URL）。这条链中没有断点。

使用 `templates/report-template.md` 作为报告结构。以下是关键要求：

### 三层编号体系

编号在 Phase 2 和 Phase 3 中随工作推进自然产生——收集来源时分配 [S 编号]，产生发现时分配 F 编号。Phase 4 将它们带入报告，不需要重新编号。章节以名称定位，不使用编号。

- **结论段** →「研究结论」章节，按 Phase 0 目标逐段组织，每段以 `← F编号` 标注依据的发现
- **来源编号** `[S1]` `[S2]` ... →「资料来源」和「参考资料」章节，附带完整 URL 和摘要
- **发现编号** `F1` `F2` ... →「交叉比对」章节，每个 F 标注依据的 S 编号

### 资料摘要的独立性

每一条来源的摘要必须**具体、完整**——保留数字、日期、引述、原文的关键论点。摘要不能是笼统描述（"讨论了某话题"）。足够详细，使读者不读原文也能理解来源说了什么。摘要是「交叉比对」章节中每条发现的依据基础——如果摘要太模糊，发现就成了无根之木。

### URL 完整性

「资料来源」和「参考资料」章节中的每条来源必须包含完整的 URL 地址。来源不可追溯 = 链条在最后一步断裂。如果某条信息确实无法回溯到 URL（如线下访谈、内部文档），标注获取方式和局限性。

### 空白诚实

Phase 3 中标记的"单来源"事实、"来源不明"数据、"不可评估"的经验——全部保留在报告中。不要为了报告的整洁而删掉"这还没确认"的信息。

### 未采用来源

Phase 2 中收集过但 Phase 3 判定不纳入结论的来源（冗余、低可靠性、不相关等），在报告中以简短列表记录——编号、标题/URL、一行排除原因。不写完整摘要。这既是诚实（展示未挑拣来源），也避免将来重复评估。

### 后续建议

基于 Phase 3 发现的遗留问题，建议下一步可以追踪的方向（如果有）。

### 交付前完整性验证

在报告完成前，逐条检查：

- 「研究结论」章节的每一条结论 → 是否能追溯到至少一条「交叉比对」章节的发现？
- 「交叉比对」章节的每一条发现 → 是否标注了依据的 [S 编号]？
- 被引用的 [S 编号] → 在「资料来源」章节中是否有完整的摘要？
- 「资料来源」章节中的每条来源 → 在「参考资料」章节中是否有 URL？
- 是否存在"来源丢失但信息仍被作为结论支撑"的情况？（如果有，要么标注为不可靠，要么回到 Phase 2 补全）

## Quality Checklist

研究完成前逐条自检：

- [ ] **指称确认**：最终研究使用的名称是否正确？Phase 1 修正记录是否保留？
- [ ] **目标覆盖**：Phase 0 拆解出的每个目标都有信息覆盖？
- [ ] **来源追溯**：每条关键信息都标注了出处和 URL？来源丢失的信息是否诚实标注？
- [ ] **追踪链完整**：每条结论 ← 发现 ← 依据（来源摘要）← URL 的链条是否有断点？是否存在引用不存在的 [S 编号]、[F 编号]？是否存在有结论无发现、有发现无来源的情况？
- [ ] **事实置信度**：关键事实是否经过交叉比对？单来源事实是否标注？
- [ ] **数据溯源**：引用的数据是否可以追溯到测量方法和定义边界？来源不明的数据是否标注？
- [ ] **知识系统性**：收集到的框架/知识是否被系统化组合而非简单罗列？框架之间的关系是否被分析？
- [ ] **意见争议**：是否存在实质性意见分歧？是否分析了争议的本质（前提/价值/信息差异）？
- [ ] **矛盾深追**：高权威来源之间的事实矛盾是否作为新问题回到了 Phase 2 追踪？
- [ ] **具体性**：收集到的是具体信息（数字、日期、引述），而非笼统概括？
- [ ] **范围锚定**：最终结论是否仍在 Phase 0 确定的范围内？超出的部分是否显式标注？
- [ ] **边界诚实**：仅依据摘要的信息、未经确认的推断——是否标注？
- [ ] **资料独立于结论**：报告中每条来源的摘要是否足够完整（读者不看结论也能理解）？结论是否可追溯到具体资料编号？
- [ ] **URL 完整**：报告中每条可追溯的来源是否包含了 URL 地址？
