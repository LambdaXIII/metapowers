## 五、模拟过程中的关键决策点

### 5.1 消歧中的"电影"前缀处理
用户手动输入了"**电影**《奥本海默》"而非单纯"奥本海默",这意味着用户已经帮 Agent 完成了最重要的消歧工作。Agent 应当**识别到用户已缩小范围**,无需列出候选人/物理学家诺兰的候选,直接确认指代是电影。

### 5.2 分类时人物模板与作品模板的区分
"奥本海默"这个名称本身就指向一个历史人物（J. Robert Oppenheimer）,但用户的查询限定在"电影《奥本海默》",因此实体是**以该人物命名的电影作品**,而非该人物本人。分类时依据的是搜索结果**针对电影**出现的线索（导演、发行日期、类型标签）,而非针对人物的线索（生卒日期、职业身份）。

### 5.3 一句话地位的推导过程
```
综合判断链：
  创作者（诺兰——作者导演标杆）
  + 年份（2023——疫情后影院复苏期）
  + 类型（传记惊悚片——严肃题材商业片）
  + 市场表现（$9.52亿 + 奥斯卡7奖 + 豆瓣8.8/IMDb 8.3）
  = 一句话地位："商业与艺术罕见平衡的当代史诗"
```
这是**自底向上的归纳推理**,而非用搜索词搜索"一句话地位"。

### 5.4 置信回检的"抽样"纪律
回检中发现"诺兰票房最高作品"这一新信息,按技能规定"发现更新的信息 → 更新该维内容",将其补充到了维度5。但**不因此重新打开搜索链路**去深追"诺兰票房排名"——符合薄框架设计。
