# Tags

> 本文档是 journal 的 tag 受控词汇表。所有条目使用的标签必须来自此表。
> 这是一个成熟 tag 集示例（约 20 个 tag，按维度分组）。初始种子只有 6 个 tag，新 tag 随内容需求而增长。

## Rules

- 每条条目至少选一个 tag
- project tag 可选，但如有明确项目归属则推荐添加

## Tags

### Activity

| Tag | Meaning |
|-----|---------|
| `design-decision` | 设计决策、架构选择 |
| `research` | 调研、信息搜集与分析 |
| `debugging` | 调试、问题排查 |
| `migration` | 环境/技术栈/数据迁移 |
| `implementation` | 实现、编码落地 |
| `plan` | 计划制定、路线图 |
| `documentation` | 文档编写、修缮 |
| `review` | 审查、评估、验收 |

### Domain

| Tag | Scope |
|-----|-------|
| `architecture` | 系统架构、组件设计 |
| `frontend` | 前端/UI |
| `tool-system` | 工具链、开发工具 |
| `workflow` | 工作流程、方法论 |
| `environment` | 运行环境、操作系统 |

### Meta

| Tag | Meaning |
|-----|---------|
| `journal` | Journal 自身维护 |
| `skill` | Skill 开发/维护 |
| `methodology` | 方法论提炼、范式总结 |
| `lesson` | 教训、经验总结 |

### Project

| Tag | Scope |
|-----|-------|
| `scriptum` | 项目 A 相关 |
| `metapowers` | Metapowers 项目 |
| `hermes-ops` | Hermes 运维相关 |
