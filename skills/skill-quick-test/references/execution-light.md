# Light 模式执行

> 📍 本文件是 skill-quick-test skill 的参考文件。

---

## 适用条件

复杂测试场景 < 2 且总测试数 < 5，且用户已确认场景设计。

---

## 子代理委派格式

每个测试单元委派一个独立子代理。所有单元并行执行。

### 技能加载单元的 Context 模板

加载技能的单元使用以下模板。构造时替换 `[skill-path]` 为被测技能的绝对路径。

> **入口路径仅适用于加载技能的单元。** baseline 单元使用下方的 Baseline Context 模板，无入口路径字段。

```
你正在进行一次技能文档的模拟使用测试。

你的目标不是「完成文档里教授的任务」，而是「检验文档本身是否清晰、完整、正确」。

硬约束:
  - 禁止 web_search、web_extract、browser_navigate
  - 禁止任何联网操作。禁止调用外部 API。
  - 仅允许：read_file（读取技能文件）、write_file（仅 Full 模式时允许）

使用场景:  ← 可选（边界测试/错误模式测试可省略整节）
  需求明确度: [精确 / 抽象 / 模糊]
  目标类型: [生成 / 查询 / 诊断]
  背景上下文: [裸 / 半知 / 全知]

模拟背景:  ← 可选
  [主代理准备：给子代理足够的领域知识来推演决策，替代真实搜索。
   描述用户知道什么、不知道什么、有什么材料、想达成什么——
   让子代理能据此判断技能指引是否清晰。]

入口路径:  ← 可选（仅适用于有明确入口的复杂场景）
  起点: [具体文件路径]
  来源方式: [正门 / 侧门 / 回访 / 引用跳转]

用户想达成什么:
  [用自然语言描述。不告知具体操作路径或预期操作。]

加载规则:
  - 被测技能：使用 read_file 读取 [skill-path]/SKILL.md
  - 被测技能的子文件：同样用 read_file，按 SKILL.md 中的相对路径拼接
  - 禁止对被测技能使用 skill_view 加载
  - 如果被测技能文件读取失败，返回「推演中止」报告

推演指引:
  - 清空：假装你从未读过这份文档。不带任何预设结论进入。
  - 逐步骤跟随：不跳过、不编造。遇到模糊/矛盾/缺失的指引 → 记录为困惑点，继续推演。
  - 边界刨：故意走至少两条异常路径（如跳过某个章节、误解某个术语、从中间进入）。
  完成后输出报告。

报告格式（硬约束，不可偏离）:
  - **必须**使用以下 Markdown 格式输出。禁止用 JSON、纯文本或任何其他格式。
  - 每个章节（路径追踪、停顿/困惑/误解、断裂点、意外发现）都必须出现，即使内容为空也要保留章节标题。
  - 缺失的章节 = 报告不合格。

  # 测试报告：[你的场景 ID]-[你的单元 ID]
  **使用场景:** [复述] | **入口路径:** [起点, 来源] | **用户想达成:** [复述]
  ## 路径追踪
  1. [文件 A] → [文件 B]：原因
  [如有能力模拟，标注: [能力模拟] [委派/工具/脚本] → 动作 → 推演结果]
  ## 停顿/困惑/误解
  | # | 文件:行号 | 类型 | 描述 |
  ## 断裂点
  | # | 文件:行号 | 类型（死链/锚点/术语）| 描述 |
  ## 意外发现
  [不在预期范围的发现]
  ## 推演中止（如有）
  [仅当系统性障碍导致无法继续时填写，替代上面所有正常节]

  > 注意：你不要打分。打分需要全局对比视角，由主代理在汇总阶段统一进行。

失败行为:
  如果遇到无法继续的障碍，不要尝试绕道自救。
  直接输出结论为「推演中止」的报告，描述障碍的具体位置和原因。
  - 被测技能文件读取失败
  - 所有入口路径均断裂
  - 其他系统性障碍（非暂时的）

能力模拟:
  被测技能中涉及的外部能力，不实际调用——模拟执行并记录假设。

  1. 子代理委派（delegate_task / task）：
     不能实际委派（深度限制）。自己扮演被委派的角色，按任务目标继续推演。
  2. 工具调用（bash、eval、browser 等）：
     不在被测项目上真实执行。读取技能对工具的使用意图，推演合理结果。
  3. 脚本执行（scripts/ 下的脚本）：
     不实际运行。用 read_file 读取脚本源码，推演其逻辑输出。

  路径追踪中统一标注：[能力模拟] [委派/工具/脚本] → 动作描述 → 推演结果

约束:
  禁止写入任何文件。所有发现通过返回值报告。
```

### Baseline Context 模板

Baseline 单元不加载技能。Agent 没有入口路径、没有文件可读——它只能用通用知识尝试完成任务。构造时无需替换 `[skill-path]`。

```
你正在进行一次技能文档的基线对照测试。

你的目标：在**不加载被测技能**的情况下，尝试完成用户的任务。
记录你实际的决策过程和遇到的困难。

硬约束:
  - 禁止 web_search、web_extract、browser_navigate
  - 禁止任何联网操作。
  - 仅允许：read_file、write_file（仅 Full 模式时允许）
  - 禁止读取被测技能目录下的任何文件
  - 禁止尝试定位或枚举被测技能的文件位置

使用场景:  ← 可选
  需求明确度: [精确 / 抽象 / 模糊]
  目标类型: [生成 / 查询 / 诊断]
  背景上下文: [裸 / 半知 / 全知]

模拟背景:
  [描述 agent 已知什么、不知道什么。
   注意：baseline agent 不知道被测技能的任何文件或结构。]

用户想达成什么:
  [用自然语言描述。不告知具体操作路径。]

推演指引:
  - 基于你的通用知识，规划如何响应用户的需求
  - 记录你的思考过程：你会先做什么、后做什么、为什么
  - 记录遇到的困惑：没有技能指引时，哪些决策让你犹豫
  - 不需要真的执行——只记录你「会」怎么做和「为什么会这样做」
  完成后输出报告。

报告格式（硬约束，不可偏离）:
  - **必须**使用以下 Markdown 格式输出。禁止用 JSON、纯文本或任何其他格式。
  - 每个章节（决策路径、困惑/犹豫、断裂点、与有技能时可能不同的地方）都必须出现，即使内容为空也要保留章节标题。
  - 缺失的章节 = 报告不合格。

  # Baseline 报告：[你的场景 ID]-[你的单元 ID]
  **使用场景:** [复述] | **用户想达成:** [复述]
  ## 决策路径
  1. [你会先做什么] → 原因
  2. [然后做什么] → 原因
  ## 困惑/犹豫
  | # | 维度 | 困惑描述 |
  ## 断裂点
  [没有技能指引时无法继续的地方]
  ## 与有技能时可能不同的地方
  [你推测有方法论指引时结果会怎样不同]

  > 注意：你不要打分。

Full模式:
  允许写入: <temp-dir>/reports/<unit-id>.md
  报告内容：模拟决策路径、困惑点、与有技能指引时可能不同的地方。
```

### 禁止事项

以下信息**绝对不能**出现在委派提示中：

- 预期效果
- "这是测试场景 S03 的 baseline 单元"
- "其他子代理也在平行测试"
- 任何形式的对比或预期引导

---

## 并发控制

- 所有测试单元同批委派，使用 `delegate_task` 的 `tasks` 数组模式
- 如果总单元数 > 最大并发数，分多批
- 等待全部子代理完成后再汇总

---

## 委派后中止检查

全部子代理返回后、汇总之前，先检查两类致命错误：

### A. 启动失败（spawn failure）

子代理未能成功启动（如配额错误 429、基础设施错误、超时等）：

1. 统计启动失败的子代理数量
2. 如果 **≥ 2 个**子代理启动失败：
   - **中止本轮测试**
   - 向用户报告：失败数量 + 错误类型 + 建议（检查配额/重试/减少并发）
   - 不进入汇总
3. 如果只有 1 个启动失败：
   - 该单元标记为 ❌，计入汇总
   - **继续**其他单元的汇总

### B. 推演中止（runtime abort）

子代理成功启动但返回「推演中止」结论：

1. 统计返回「推演中止」结论的子代理数量
2. 如果 **≥ 2 个**子代理中止，且障碍是**同一类系统性问题**（如被测技能路径错误、文件格式导致所有子代理无法读取）：
   - **中止本轮测试**
   - 向用户报告诊断：系统性问题 + 中止数量 + 障碍描述
   - 不再进入汇总
3. 如果只有 1 个子代理中止，或中止原因各自不同且非系统性：
   - 该子代理的报告标记为 ❌，计入汇总
   - **继续**其他子代理的汇总
