# 行为定义型框架（Behavior Definition Framework）

## 适用场景

**特征**：
- 需要自主决策（不是被动执行）
- 需要使用工具或调用其他智能体
- 需要根据情况灵活调整
- 任务边界不完全清晰（需要判断）

**典型智能体**：
- 专家顾问（市场分析、财务规划、技术评估）
- 调度决策者（任务分配、资源协调）
- 研究分析师（需要判断信息价值、选择分析方法）

## 结构模板

```markdown
---
description: "简短描述，用于Agent选择"
mode: "all"  # 或其他适用模式
---

**一句话定位**：你是[角色]，用[方法/视角]将[输入]转化为[产出]。

## 核心方法（必须详细设计）

### [方法类别]
**[具体方法名称]**
- **[子方法1]**：[如何运用，解决什么问题，判断标准]
- **[子方法2]**：[如何运用，解决什么问题，判断标准]
...

### [方法类别]
...

## 质量标准

- [质量要求1]
- [质量要求2]
...

## 你不做什么

- 不做[某事] —— 那是[某范围/某智能体]的[职责/专业]
- 不[某行为] —— 原因/后果
...

## 输出要求

必须包含：
- [要素1]
- [要素2]
...

质量自检：
- [检查问题1]？
- [检查问题2]？
...
```

## 与任务流水线型的关键差异

| 维度 | 行为定义型 | 任务流水线型 |
|------|-----------|-------------|
| **决策权** | 高（需要自主判断） | 低（按步骤执行） |
| **方法描述** | 思维模式（如何思考、如何选择） | 步骤清单（做什么、按什么顺序） |
| **灵活性** | 高（根据情况选择方法） | 低（固定流程） |
| **输出类型** | 分析结论、决策建议 | 执行结果、处理输出 |

## 方法设计要点

**必须避免**：

❌ **任务清单模式**：
```markdown
你需要完成：
1. 进行PEST分析
2. 应用五力模型
3. 评估价值链
```

✅ **思维模式模式**：
```markdown
**系统化分析框架运用**

- **宏观扫描**：运用PEST框架识别政治、经济、社会、技术四大力量的交互作用
  * 政治因素如何影响行业准入门槛？
  * 经济周期与技术变革的关联是什么？

- **竞争结构判断**：运用波特五力评估行业利润空间和竞争强度
  * 新进入者的真实威胁是什么？
  * 供应商议价能力的临界点在哪里？
```

**关键区别**：
- 任务清单说"做PEST分析"（what）
- 思维模式说"运用PEST框架识别..."（how + why）

## 使用提示

1. **方法设计是核心**：框架只提供结构，步骤2的方法设计决定质量
2. **必须回答"如何选择"**：当有多种方法时，如何决定用哪种？
3. **边界约束很重要**：明确"不做什么"防止越界
4. **质量标准必须具体**：不是"高质量"，而是"满足X条件"
