# 功能分解（Functional Decomposition）

> 自顶向下拆解复杂系统：核心能力 → 主要功能 → 子功能

---

## 核心概念

```
核心能力
├── 主要功能 A
│   ├── 子功能 A1
│   ├── 子功能 A2
│   └── 子功能 A3
├── 主要功能 B
│   ├── 子功能 B1
│   └── 子功能 B2
└── 主要功能 C
    └── 子功能 C1
```

**原则**：
- 每层之间是"包含"关系
- 同层之间是"并列"关系
- 分解到可独立理解的最小单元

---

## 在技能理解中的应用

### 分析步骤

1. **识别核心能力**：技能最本质的用途是什么？
2. **拆解主要功能**：为实现核心能力，需要哪些功能模块？
3. **细化子功能**：每个主要功能包含哪些具体操作？

### 示例：cx-skill-test

```
核心能力：验证技能效果
├── 功能1：制定测试方案
│   ├── 确认被测试技能
│   ├── 准备测试场景
│   ├── 询问基线对比
│   └── 创建测试目录
├── 功能2：执行测试
│   ├── 创建输入文件
│   ├── 发送测试指令
│   └── 收集输出结果
└── 功能3：评估结果
    ├── 读取预期效果
    ├── 对比实际输出
    └── 生成评估报告
```
