# 测试案例集

用于验证 cx-llm-prompt 技能的有效性和完整性。

## 案例1：从零创建 - 市场分析Agent

**用户输入**：
```
帮我设计一个市场分析Agent的提示词，用来做行业研究和竞争分析
```

**预期执行流程**：
1. **步骤0**：识别为创建意图
   - 目标：市场分析Agent
   - 用途：持久化智能体
   - 领域：市场研究
   - 关键决策：需要自主决策（分析判断）✓，涉及工具使用（搜索）✓
   
2. **步骤1**：推荐行为定义型框架
   - 理由：需要自主决策和专业判断

3. **步骤2**：设计方法（核心验证点）
   - 不套用"思维模式"模板
   - 根据市场分析领域设计具体方法
   - 包含：信息分层→系统化分析→模式识别→SO WHAT转化
   - 自动应用：元信息清理、一句话定位、边界约束

4. **步骤3**：生成完整提示词

5. **步骤4**：综合诊断
   - 静态：检查七维（元信息✓、完整性✓、一致性✓、具体性✓...）
   - 推演：模拟典型输入（"分析新能源汽车市场"）

6. **步骤5**：输出最终提示词

**验证要点**：
- [ ] 步骤2是否设计了具体方法而非仅列工具名？
- [ ] 是否回答了"如何做"和"如何做好"？
- [ ] 元信息是否已清理？
- [ ] 一句话定位是否清晰？
- [ ] 输出是否可用？

## 案例2：诊断场景 - 模糊提示词

**用户输入**：
```
看看这个提示词有什么问题：
{你是助手，帮我处理文档。}
```

**预期执行流程**：
1. **步骤0**：识别为诊断意图
   - 终点=步骤4（仅诊断）

2. **步骤4**：综合诊断
   - 静态检查：
     - 🔴 完整性：缺少具体方法、质量标准、输出格式
     - 🟡 具体性："处理文档"过于模糊
     - 🟢 元信息：清洁
   - 推演：
     - 输入"处理这份报告"→不知道具体做什么

3. **步骤5**：识别终点=步骤4，输出诊断报告
   - 不执行修复
   - 提供问题清单和修复方向

**验证要点**：
- [ ] 是否正确识别为诊断意图？
- [ ] 诊断报告是否准确指出问题？
- [ ] 是否没有自动修复？
- [ ] 是否提供了修复方向？

- [ ] 是否正确识别元信息问题？
- [ ] 是否自动执行修复？

## 案例6：方法设计质量验证

**测试目标**：验证步骤2不预设解原则

**用户输入**：
```
帮我设计一个财务规划Agent
```

**验证标准**：

步骤2的输出不应是：
❌ 模板化的"思维模式"：
```markdown
## 核心能力
**系统化分析框架运用**
- 宏观扫描：PEST分析...
- 竞争结构：五力模型...
```

应该是：
✅ 基于财务领域的具体方法：
```markdown
## 核心方法

**四维财务分析**

- **盈利质量分析**
  * 收入结构：主营收入占比及变化趋势
  * 利润质量：经营性利润vs非经常性损益
  * 现金流匹配：利润与经营现金流的匹配度

- **偿债能力评估**
  * 短期：流动比率、速动比率的合理区间判断
  * 长期：资产负债率、利息覆盖倍数趋势
  * 隐性负债：表外融资、或有负债识别

- **运营效率诊断**
  * 周转率分析：存货、应收、应付周转天数
  * 资本效率：ROE分解（杜邦分析）
  * 现金周期：从采购到回款的完整周期

- **成长潜力评估**
  * 收入增长率：量价拆分
  * 投资回报：ROIC与WACC比较
  * 可持续性：增长是否依赖过度杠杆
```

**验证要点**：
- [ ] 是否基于财务领域设计？
- [ ] 是否描述了具体运用方法？
- [ ] 是否仅列工具名？
- [ ] 是否回答了"如何做"和"如何做好"？

4. **问题记录**：记录偏差和问题
