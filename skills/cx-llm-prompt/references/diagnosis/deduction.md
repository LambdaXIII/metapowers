# 推演验证方法

步骤4的动态检查部分，通过模拟执行验证提示词的可行性。

## 核心思想

**推演** = 模拟Agent按提示词执行，预判可能的问题

不是真正执行，而是在脑海中"跑一遍"：
1. 如果收到这样的输入，Agent会怎么做？
2. 执行路径清晰吗？
3. 边界情况如何处理？
4. 有潜在冲突吗？

## 推演步骤

### 步骤1：准备测试用例

设计2-3个典型测试场景：

**场景A：正常输入**
- 输入：[最典型的用户请求]
- 预期：Agent应该[如何响应]

**场景B：边界输入**
- 输入：[边界情况，如空输入、超长输入、模糊请求]
- 预期：Agent应该[如何处理]

**场景C：异常输入**
- 输入：[异常情况，如恶意输入、无关输入]
- 预期：Agent应该[如何应对]

### 步骤2：模拟执行

对每个场景，按提示词逐步推演：

```
输入：[测试输入]
  ↓
Agent理解输入 → 是否符合提示词的输入规范？
  ↓
Agent选择方法/步骤 → 是否明确知道该用哪种方法？
  ↓
Agent执行 → 每个步骤是否清晰可执行？
  ↓
Agent生成输出 → 是否符合输出要求？
  ↓
结果评估 → 是否满足用户预期？
```

### 步骤3：识别问题

推演中可能发现的问题：

**路径问题**：
- 不知道该走哪条路（方法选择不明确）
- 走到一半卡住了（步骤不清晰）
- 走进了死胡同（缺少处理分支）

**判断问题**：
- 无法判断是否完成（缺少完成标准）
- 无法判断对错（缺少质量标准）
- 无法判断边界（边界描述模糊）

**输出问题**：
- 输出不符合要求（输出规范不清晰）
- 输出质量不可控（缺少质量标准）

**冲突问题**：
- 指令A和指令B矛盾
- 约束过于严格导致无法执行
- 能力描述与边界约束冲突

## 推演示例

**提示词片段**：
```markdown
你是市场分析助手，帮助用户分析市场趋势。

## 核心方法
你需要进行市场分析，提供有价值的洞察。

## 输出要求
输出一份市场分析报告。
```

**推演发现的问题**：

**场景A：正常输入**（"帮我分析一下新能源汽车市场"）
- 理解：✓ 明确
- 方法选择：✗ "进行市场分析"太模糊，不知道具体用什么方法
- 执行：✗ 没有具体步骤，无从下手
- 输出：✗ "分析报告"太笼统，不知道写什么内容

**问题总结**：
- 方法描述过于抽象，没有具体可执行的分析框架
- 缺少步骤指导
- 输出规范不明确

## 问题分级

| 等级 | 定义 | 示例 |
|------|------|------|
| 🔴 严重 | 会导致执行失败或产生严重错误 | 完全不知道怎么做、关键步骤缺失 |
| 🟡 建议 | 可能影响质量或用户体验 | 步骤不够清晰、缺少边界处理 |
| 🟢 优化 | 可以更好但当前可用 | 可以增加更多示例、可以优化结构 |

## 与静态检查的区别

| 维度 | 静态检查 | 推演验证 |
|------|---------|---------|
| **方式** | 检查文本本身 | 模拟执行过程 |
| **关注点** | 结构、完整性、一致性 | 可执行性、路径清晰度 |
| **发现的问题** | "缺少X要素" | "这样做会失败" |
| **互补关系** | 检查"有什么" | 验证"能否用" |

## 输出格式

```markdown
## 推演验证结果

### 场景1：正常输入
**输入**：[测试用例]
**执行路径**：[逐步推演]
**结果**：[输出预期]
**评估**：✅ 正常 / ⚠️ 问题：[描述]

### 场景2：边界输入
**输入**：[测试用例]
**执行路径**：[逐步推演]
**结果**：[输出预期]
**评估**：✅ 正常 / ⚠️ 问题：[描述]

### 场景3：异常输入
**输入**：[测试用例]
**执行路径**：[逐步推演]
**结果**：[输出预期]
**评估**：✅ 正常 / ⚠️ 问题：[描述]

### 潜在问题汇总
🔴 严重：[N] 个
1. [问题描述] - 影响：[什么情况下会出问题] - 建议：[修复方向]

🟡 建议：[N] 个
...
```

## 推演技巧

1. **不要假设Agent"知道"**：如果提示词没明说，Agent就不知道
2. **关注决策点**：每一步需要判断的地方，是否有明确标准？
3. **测试边界**：正常情况通常能过，边界情况容易暴露问题
4. **保持客观**：不要脑补提示词没写的内容

## 常见推演发现的问题

- **方法选择模糊**：多种方法时，不知道选哪个
- **缺少完成标准**：不知道做到什么程度算完成
- **边界处理缺失**：遇到预期外情况不知道怎么办
- **输出格式不清**：知道要输出，但不知道具体格式
- **质量标准缺失**：不知道什么算"好"的结果
