# 五维打分标准

> 📍 本文件是 [skill-quick-test skill](../SKILL.md) 的参考文件。主入口和决策树请见 SKILL.md。

---

## 定位

打分是**可选模块**，基于汇总结果进行，用于：
- 跨场景的直观质量评估
- 多轮迭代的进展追踪（Round 1 → Round 2 → ...）

打分不是逐场景必出项。默认不启用，调用者明确要求时才进行。

---

## 默认五维度

这些维度适用于大多数 Skill 的用户体验评估：

### 维度 1：全流程可执行性 ★1-5

用户能否按 Skill 的指引独立完成任务？

| 分数 | 标准 |
|:--:|------|
| ★5 | 主要 Persona 在所有入口下都能无外部帮助完成目标 |
| ★4 | 大部分完成任务，偶有轻微停顿 |
| ★3 | 能完成但需要较多试错或回退 |
| ★2 | 部分入口下用户迷路或无法完成 |
| ★1 | 多数入口下用户阻塞 |

### 维度 2：模块流转清晰度 ★1-5

文件间的导航（决策树、"下一步"、交叉引用）是否流畅？

| 分数 | 标准 |
|:--:|------|
| ★5 | 所有导航链路畅通，无环路，无歧义 |
| ★4 | 导航基本流畅，偶有冗余或方向模糊 |
| ★3 | 部分"下一步"区块指向不明确 |
| ★2 | 用户需要自行猜测跳转方向 |
| ★1 | 用户反复迷路或陷入循环 |

### 维度 3：术语全局统一 ★1-5

关键概念在全文中的命名和引用是否一致？

| 分数 | 标准 |
|:--:|------|
| ★5 | 所有关键术语在全文无变体，引用一致 |
| ★4 | 偶有不一致但不影响理解 |
| ★3 | 存在多处不一致，可能造成困惑 |
| ★2 | 核心概念有歧义 |
| ★1 | 术语混乱，同一概念多种称呼 |

### 维度 4：覆盖完整性 ★1-5

Skill 是否覆盖了其声称支持的所有场景和 Persona？

| 分数 | 标准 |
|:--:|------|
| ★5 | 所有声称的功能场景均有完整指引 |
| ★4 | 主路径完整，边缘路径偶有缺失 |
| ★3 | 存在明显的覆盖缺口 |
| ★2 | 多个功能场景缺乏指引 |
| ★1 | 核心场景有遗漏 |

### 维度 5：异常容错 ★1-5

当用户不按预期路径使用时，Skill 是否能纠偏？

| 分数 | 标准 |
|:--:|------|
| ★5 | 侧门/回访入口有自标识和返回主路径的指引 |
| ★4 | 大部分异常路径有 fallback |
| ★3 | 侧门入口有指引但不够明显 |
| ★2 | 跳读用户容易丢失上下文 |
| ★1 | 偏离预期路径即完全迷失 |

## 维度定制

如果被测技能的性质差异较大，可以替换或增减维度。

**替换原则：**
- 替换后的维度名称和标准必须明确写清楚
- 在测试计划的「打分」部分注明定制维度

**示例（代码生成 Skill 的定制维度）：**

| 维度 5（替换） | 标准 |
|---------------|------|
| 代码可编译性 | Skill 生成的代码是否能直接编译运行？ |

---

## 加权方案

默认使用算术平均（五维度等权）。如果不同 Persona 的重要性不同，可使用加权：

| Persona | 权重 | 理由 |
|---------|:--:|------|
| 新手·空白 | 1.5× | 如果 Skill 的目标用户主要是新手 |
| 高级·侧门 | 0.5× | 如果核心用户路径是正门 |

加权方案在测试计划中声明。

```
全流程可执行性: ★★★★☆ (4.2/5)
模块流转清晰度: ★★★★★ (5.0/5)
术语全局统一:   ★★★★☆ (4.5/5)
覆盖完整性:     ★★★★☆ (3.8/5)
异常容错:       ★★★☆☆ (3.0/5)
─────────────────────────
全局加权平均:   ★★★★☆ (4.1/5)  [↑0.6 vs Round 2]
```
