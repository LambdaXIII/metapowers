# 角色驱动型框架（Role-Driven Framework）

## 适用场景

**特征**：
- 需要与用户高频交互
- 需要建立关系、理解语境
- 需要情感智能
- 任务目标需要通过对话逐步明确

**典型智能体**：
- 客户服务助手
- 教育辅导助手
- 心理咨询助手
- 虚拟伴侣

## 结构模板

```markdown
---
description: "简短描述，用于Agent选择"
mode: "all"
---

**一句话定位**：你是[角色]，用[风格/方式]提供[价值]。

## 人格设定

- **核心特质**：[3-5个关键词，如：耐心、专业、友善]
- **语气风格**：[正式/轻松/幽默/严肃等]
- **价值观**：[优先考虑的，如用户满意、安全、成长等]

## 交互协议

### 开场
- [首次交互时的开场方式]

### 理解阶段
- **意图识别**：如何理解用户的真实需求？
- **语境理解**：如何记住和利用对话历史？
- **情感识别**：如何识别用户情绪状态？

### 回应阶段
- **回应策略**：
  - 当用户[情况A]时，[如何回应]
  - 当用户[情况B]时，[如何回应]
- **升级规则**：何时需要转人工/升级处理？

### 结束阶段
- [如何优雅结束对话]
- [是否需要总结/跟进]

## 边界约束

- **安全边界**：什么内容绝不回应？
- **能力边界**：什么请求无法满足？
- **伦理边界**：什么行为必须拒绝？

## 记忆策略

- **短期记忆**：当前对话的哪些信息需要记住？
- **长期记忆**：跨对话的哪些信息需要保留？

## 质量自检

- [ ] 是否保持了人设一致性？
- [ ] 是否理解了用户意图？
- [ ] 回应是否符合用户情绪状态？
- [ ] 是否提供了有价值的帮助？
```

## 与其他框架的关键差异

| 维度 | 角色驱动型 | 行为定义型 | 任务流水线型 |
|------|-----------|-----------|-------------|
| **核心** | 人格 + 交互 | 决策 + 方法 | 流程 + 输出 |
| **交互模式** | 多轮对话 | 可能单轮 | 通常单轮 |
| **关系建立** | 重要 | 次要 | 不需要 |
| **情感智能** | 需要 | 不需要 | 不需要 |

## 设计要点

1. **人格要具体**：不是"友善"，而是"像经验丰富的老师一样耐心"
2. **交互规则要场景化**：不是"礼貌回应"，而是"当用户生气时，先道歉再解决"
3. **边界要清晰**：什么能做、什么不能做，要在对话中保持一致
4. **记忆策略要实用**：不要为了记忆而记忆，记住真正有用的信息

## 常见陷阱

- **人设模糊**：说"友善"但没有具体定义
- **规则死板**：不考虑语境，机械执行规则
- **过度热情**：不适当的情感表达让人感到不适
- **边界不清**：今天能做，明天不能做，让用户困惑
