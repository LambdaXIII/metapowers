# 渐进式披露（Progressive Disclosure）

> L1概览 → L2摘要 → L3详情 → L4深度

---

## 核心概念

| 层级 | 内容深度 | 阅读时间 | 目标读者 |
|------|----------|----------|----------|
| **L1 概览** | 一句话定义 + 适用场景 | 30秒 | 所有人 |
| **L2 摘要** | 核心概念 + 主要流程 | 2分钟 | 潜在用户 |
| **L3 详情** | 完整流程 + 决策规则 | 10分钟 | 实际使用者 |
| **L4 深度** | 设计思想 + 版本历史 | 按需 | 维护者/开发者 |

**原则**：不要一次性展示所有信息，让用户按需深入。

---

## 在技能文档中的应用

### 输出结构映射

```
<output-dir>/
├── README.md              # L1: 概览层
│   ├── 一句话定义
│   ├── 适用场景
│   ├── 快速开始
│   └── 文档导航
├── methodology/           # L2-L3: 方法论层
│   ├── core-concepts.md   # L2: 核心概念（Explanation）
│   ├── workflow.md        # L3: 工作流程（How-to）
│   └── decision-rules.md  # L3: 决策规则（Reference）
├── reference/             # L3: 参考层
│   ├── guide-xxx.md       # L3: 精炼参考
│   └── examples.md        # L3: 典型示例
└── meta/                  # L4: 深度层
    ├── changelog.md       # L4: 版本历史
    └── design-notes.md    # L4: 设计笔记
```

### 设计原则

- **L1 必须独立**：只读 README.md 就能判断"这个技能是否适合我"
- **L2 建立认知**：理解核心概念后，能判断"我是否需要深入了解"
- **L3 支持执行**：有完整信息支持实际使用
- **L4 供查阅**：需要时深入，不需要时不干扰

---

## 检查清单

- [ ] L1 是否能在30秒内读完？
- [ ] L1 是否包含"是什么"和"适合谁"？
- [ ] L2 是否解释了核心概念？
- [ ] L3 是否支持实际执行？
- [ ] L4 是否不影响前层的简洁性？
