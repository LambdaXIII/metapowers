# DESIGN

> 本文档记录 cx-skill-test 技能的设计意图和核心决策，作为维护参考。

## 设计初衷

解决技能开发过程中的验证问题——需要快速确认技能能否正确工作，但不追求全面覆盖。

## 核心思想

**快速验证，结构化执行，可选增强**

- **快速**：单次验证，快速迭代
- **隔离**：测试阶段不安装到运行时环境，使用独立 SubAgent
- **真实**：使用真实场景，不虚构用例
- **可复现**：通过 eval_metadata.json 记录测试信息

## 关键决策

| 决策 | 理由 |
|------|------|
| 四阶段流程 | 结构化确保测试可复现，避免遗漏关键步骤 |
| 两种 SubAgent | 测试执行用领域匹配 SubAgent，评估用 general 保证客观 |
| 场景 5 步法 | 融入技能分析到场景准备，确保覆盖核心能力 |
| 基线对比可选 | 不是每次都需要，按需启用更灵活 |
| 评估用描述性 expected | 避免主观判断，提供可验证的依据 |

## 与 skill-creator 的关系

| 维度 | cx-skill-test（快速流程） | skill-creator（标准流程） |
|------|--------------------------|--------------------------|
| 核心目标 | 验证是否符合预期 | 测量改进程度 |
| 评估方式 | 主观判断 + 可选断言 | 量化数据 + 主观反馈 |
| 适用阶段 | 开发迭代 | 正式发布前 |
