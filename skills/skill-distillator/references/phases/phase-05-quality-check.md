# 第五阶段：质量检查

> 交付前自我评估，确保输出文档既忠实于原技能，又对人类读者友好。

---

## 进入条件

- 前置输出：phase-04 生成的所有输出文件
- 前置输出：phase-03 的文档大纲（含输出形态判定和文件清单）
- 确认：phase-04 `阶段完成确认` 已通过

## 目标

通过系统性检查和沉浸式推演，验证输出文档的质量。

### 形态合规（致命——不通过即回退）

| 检查项 | 方法 |
|--------|------|
| 输出形态是否符合 phase-03 判定？ | 对比实际输出文件数与 phase-03 文件清单 |
| 多文件判定时，是否实际生成了所有文件？ | 逐文件对照 phase-03 文件清单 |
| 是否中途从多文件降级为单文件？ | 如实际为单文件但 phase-03 判定为多文件 → 致命错误，回退到 phase-04 |

形态合规检查必须最先执行。不通过 → 终止质量检查，回退到对应阶段。

## 检查维度

| 维度 | 检查内容 |
|------|----------|
| **完整性** | 是否生成了所有应有的文件？必要章节是否存在？ |
| **准确性** | 内容是否忠实于原技能？有无误解或遗漏？ |
| **可读性** | 人类能轻松理解吗？视角转换完成了吗？ |
| **一致性** | 术语、格式、风格是否统一？ |
| **渐进式披露** | L1-L4 层级是否清晰？读者能按需深入吗？ |
| **智能转换** | 清晰的内容是否直接摘取？指令型内容是否转为说明型？简略内容是否丰富？ |
| **风格指南** | 是否使用主动语态、第二人称、现在时态？ |
| **源一致性** | 输出文档是否正确处理了源文件之间的矛盾？裁决策略是否合理？ |
| **身份保真** | 输出是否描述技能本身的方法论（而非技能处理的主题）？每文件独立检查。发现主题漂移 → 回退到 phase-04 |
| **语言一致性** | 输出语言是否与 phase-01 记录的语言标签一致？ |

## 类型自适应检查

根据技能类型，增加针对性检查：

纯指令型技能：
- [ ] 推断内容是否合理？是否标注 `[推断: 依据]`？
- [ ] 检查清单是否按"定义+检测方法"模式转换？
- [ ] 决策规则表是否按"场景-行为-理由"格式转换？
- [ ] core-concepts.md 是否合并到 workflow.md（而非生成空文件）？

混合型技能：
- [ ] 已清晰的内容是否直接摘取，未过度转换？
- [ ] 指令型内容是否完成视角转换？

## 检查方法

### 方法一：对比检查

对照原始技能文件，验证输出内容的准确性。

- 核心概念是否忠实？
- 工作流程是否准确？
- 决策规则是否完整？
- 约束条件是否保留？

### 方法二：推演验证（沉浸式模拟阅读）

Agent 扮演"第一次接触这个技能的读者"，从不同深度检查：

#### L1 检查（30秒）
只阅读 README.md：
- 能判断这个技能是否适合我吗？
- 能快速了解核心功能吗？
- 能找到文档导航吗？

#### L2 检查（2分钟）
阅读 README.md + methodology/core-concepts.md：
- 能理解核心概念吗？
- 能判断是否需要深入了解吗？
- 设计思想清晰吗？

#### L3 检查（10分钟）
完整阅读 methodology/ 和 reference/：
- 能理解执行逻辑吗？
- 决策规则清晰吗？
- 示例有帮助吗？

#### L4 检查（按需）
阅读 meta/：
- 设计笔记对理解有帮助吗？
- 版本历史清晰吗？

## 检查规则

1. **逐文件检查**：每个输出文件独立检查
2. **发现问题立即修复**：对照原技能修正
3. **修复后重新检查**：修改过的文件必须重新检查
4. **停止条件**：连续两轮完整检查无修改则停止
5. **最大迭代轮数**：最多 5 轮完整检查，防止无限循环
