# 四大框架总览

## 框架选择逻辑

根据需求分析中的关键决策点选择框架：

| 关键特征 | 推荐框架 | 核心强调 |
|---------|---------|---------|
| 需要自主决策 + 使用工具 | **行为定义型** | 决策逻辑和边界约束 |
| 高频交互 + 情感连接 | **角色驱动型** | 人格和交互风格 |
| 明确输入 + 固定步骤 | **任务流水线型** | 流程和输出规范 |
| 以上多种组合 | **混合架构型** | 全面覆盖各维度 |

## 决策点详解

### 1. 是否需要自主决策？

**是** → 倾向于行为定义型或混合架构型
- 需要判断"是否执行"、"如何执行"、"何时停止"
- 需要根据情况选择不同路径
- 需要处理不确定性

**否** → 倾向于任务流水线型
- 有明确的输入和处理步骤
- 不需要判断，只需执行
- 边界清晰

### 2. 是否涉及高频交互？

**是** → 倾向于角色驱动型或混合架构型
- 需要与用户多轮对话
- 需要建立关系、理解语境
- 需要情感智能

**否** → 其他框架均可

### 3. 任务边界是否清晰？

**清晰** → 倾向于任务流水线型
- 输入明确、输出明确
- 步骤可标准化
- 可重复执行

**模糊** → 倾向于行为定义型或混合架构型
- 需要灵活处理
- 需要专业判断
- 需要根据情况调整

## 框架对比

| 维度 | 行为定义型 | 角色驱动型 | 任务流水线型 | 混合架构型 |
|------|-----------|-----------|-------------|-----------|
| **核心特征** | 决策+边界 | 人格+交互 | 流程+规范 | 全面覆盖 |
| **自主程度** | 高 | 中 | 低 | 高 |
| **交互频率** | 低 | 高 | 低 | 高 |
| **复杂度** | 中 | 低 | 低 | 高 |
| **典型场景** | 专家顾问 | 客服助手 | 数据处理 | 复杂系统 |

## 关键提醒

**框架提供结构，不决定内容**

框架告诉你"提示词应该包含哪些部分"，但不告诉你"每个部分写什么"。

**内容设计（步骤2）才是关键**：
- 如何回答"如何做"？
- 如何回答"如何做好"？

框架只是骨架，方法设计才是血肉。
