# 冰山模型（Iceberg Model）

> 由表及里理解系统：事件 → 模式 → 结构 → 心智模型

---

## 核心概念

```
        ↑ 水面 ↑
    ┌─────────────┐
    │   事件      │  可见：发生了什么
    ├─────────────┤
    │   模式      │  可察：反复出现的行为
    ├─────────────┤
    │   结构      │  可推：系统如何运作
    ├─────────────┤
    │  心智模型   │  深层：为什么这样设计
    └─────────────┘
```

| 层级 | 问题 | 在技能中的对应 |
|------|------|---------------|
| 事件 | 技能做了什么？ | SKILL.md 中的具体操作步骤 |
| 模式 | 技能反复使用什么方法？ | 核心流程、常见模式 |
| 结构 | 技能如何组织？ | 文件结构、依赖关系 |
| 心智模型 | 为什么这样设计？ | DESIGN.md 中的设计思想 |

---

## 在技能理解中的应用

### 分析步骤

1. **事件层**：列出技能的所有操作（读取文件、执行步骤、输出结果）
2. **模式层**：识别反复出现的行为（如"先读取再判断"、"分阶段执行"）
3. **结构层**：分析文件组织方式、模块依赖、数据流
4. **心智模型层**：理解设计者的核心假设和价值观

### 示例：cx-skill-test

| 层级 | 内容 |
|------|------|
| 事件 | 制定方案 → 执行测试 → 评估结果 |
| 模式 | 每个测试用例独立 SubAgent、输入输出分离 |
| 结构 | 四阶段流程、两种指令、eval_metadata.json |
| 心智模型 | "盲测"原则：测试者不应知道正确答案 |
