# 一句话定位设计公式

## 核心公式

```
你是[角色定位]，[核心方法/视角]，[价值产出]。
```

## 设计原则

1. **10秒原则**：读者应在10秒内完全理解
2. **暗示方法**：通过措辞暗示工作方式
3. **明确价值**：清晰说明产出是什么
4. **区分定位**：与相似角色有明显区分

## 结构解析

| 组成部分 | 作用 | 示例 |
|---------|------|------|
| **角色定位** | 建立身份认同 | "市场洞察专家"、"财务战略伙伴" |
| **核心方法** | 暗示工作方式 | "用系统化分析框架"、"用量化思维" |
| **价值产出** | 明确交付标准 | "转化为战略洞察"、"支撑战略决策" |

## 优秀示例

### 示例1：调度决策型

**一句话定位**：
> 你是总经理（总裁），代行经营管理职责，统领专家层执行战略，对经营结果负责。

**结构分析**：
- 角色：总经理（总裁）
- 方法：代行管理、统领专家层
- 产出：对经营结果负责

### 示例2：专家分析型

**一句话定位**：
> 你是市场洞察专家，用系统化分析框架将碎片化信息转化为可指导决策的战略洞察。

**结构分析**：
- 角色：市场洞察专家
- 方法：系统化分析框架
- 产出：可指导决策的战略洞察
- **隐含信息**：输入是"碎片化信息"，暗示需要处理不确定性

### 示例3：架构维护型

**一句话定位**：
> 你是智元大师，独立于智能体办公室架构之外的观察者，专责维护智能体系统本身的架构设计与规范，对自己的存在痕迹最小化。

**结构分析**：
- 角色：智元大师（独特命名）
- 视角：独立于架构之外的观察者
- 产出：维护系统架构
- **特殊强调**："独立于...之外"明确区分于业务智能体

## 设计检查清单

完成设计后验证：

- [ ] 是否10秒内能读懂？
- [ ] 是否区分于其他智能体？
- [ ] 是否暗示了工作方法？
- [ ] 是否体现了价值产出？
- [ ] 是否暗示了输入类型？
- [ ] 一句话是否能作为所有行为的锚点？

## 常见错误

**❌ 过于冗长**：
> 你是一个专业的市场研究分析师，擅长进行行业分析和市场调研，拥有多年的研究经验...

**❌ 过于简单**：
> 你是助手。

**❌ 缺少方法或产出**：
> 你是市场分析师。

**✅ 正确示例**：
> 你是市场洞察专家，用系统化分析框架将碎片化信息转化为可指导决策的战略洞察。

## 适用场景

**建议使用**：
- 持久化智能体（长期使用的系统提示词）
- 需要明确身份认同的场景
- 复杂智能体需要快速建立定位

**可选使用**：
- 一次性任务提示词（可直接描述任务）
- 简单任务不需要身份建立

## 与其他要素的关系

一句话定位是**起点**，后续内容需要与之呼应：

```
一句话定位 → 核心方法展开 → 质量标准 → 边界约束
   ↓              ↓              ↓          ↓
"我是谁"      "我如何做"      "什么算好"   "我的边界"
```

**一致性检查**：
- 定位说"系统化分析框架" → 核心方法必须描述系统化方法
- 定位说"转化为战略洞察" → 质量标准必须要求洞察质量
