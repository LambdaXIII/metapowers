# 元信息清理指南

## 什么是元信息

描述智能体"在系统中处于什么位置"而非"如何工作"的信息。

**典型元信息**：
- "与内档的关键区别是..."
- "当需要最新信息时..."
- "作为支持层..."
- "与工正紧密配合..."
- "当你被唤起时执行..."

## 为什么必须清理

1. **认知负担**：增加不必要的认知维度
2. **决策困惑**：智能体尝试自行判断协作关系
3. **职责模糊**：暗示智能体需要判断自己是否应该被调用
4. **架构污染**：系统位置信息应集中在架构文档

## 清理原则

### 保留（执行信息）
- 你做什么（核心能力）
- 怎么做（工作方法）
- 质量标准（输出要求）
- 边界约束（你不做什么）

### 移除（元信息）
- 与他人的区别
- 触发条件
- 系统层级描述
- 协作指令

## 清理对照表

| 元信息类型 | 原文示例 | 清理后 |
|-----------|---------|--------|
| 与其他智能体的区别 | "与内档的关键区别：内档管理本地信息，你专门从外部网络获取新信息" | 移除，或在架构文档中说明 |
| 触发条件 | "触发条件：需要最新市场动态时" | 移除，由调度者决定 |
| 系统层级 | "作为支持层，你负责..." | 移除，直接描述职责 |
| 协作关系 | "与工正紧密配合，确保技术方案适配..." | 移除，只描述自己的工作 |
| 被动描述 | "当你被唤起时执行..." | 改为"你执行..." |

## 清理后对比

**清理前**：
```markdown
你是智能体办公室的外部调查员...

与内档的关键区别：
- 内档管理本地已有信息
- 你专门从外部网络获取新信息

触发条件：
- 需要最新市场动态时
- 需要竞品信息时

与工正的配合：
- 技术调研需要与技术方案对齐
```

**清理后**：
```markdown
你是外部调查员，专门从网络搜集外部信息...

## 核心工作方法
（直接描述如何执行工作，不提及他人）

## 边界约束
（只描述自己的工作边界，不描述与他人的关系）
```

## 诊断检查项

检查提示词是否包含元信息：

- [ ] 是否包含"与XX的区别"类描述？
- [ ] 是否包含"触发条件"类描述？
- [ ] 是否包含"在系统中"类定位描述？
- [ ] 是否包含其他智能体应如何调用我的指令？
- [ ] 是否包含"当你被唤起时"类被动描述？
- [ ] 是否暗示智能体需要自行判断协作关系？

如果任一答案为"是"，则需要清理。

## 安置去向

| 元信息类型 | 去向 | 理由 |
|-----------|------|------|
| 与其他智能体的区别 | AGENTS.md | 系统架构描述 |
| 触发条件 | AGENTS.md / 调用逻辑 | 由调度者决定 |
| 系统层级描述 | AGENTS.md | 架构文档已说明 |
| 协作关系 | 调度逻辑 | 由调度者在调用时管理 |

## 清理决策树

```
发现疑似元信息
    ↓
是否描述"我与其他智能体的关系"？
    ↓是→ 移至AGENTS.md
    ↓否
是否描述"我何时被调用"？
    ↓是→ 移至调用逻辑/AGENTS.md
    ↓否
是否描述"我在系统中的位置"？
    ↓是→ 移至架构文档
    ↓否
保留（可能是执行相关信息）
```
