# 金字塔原理（Pyramid Principle）

> 结论先行、以上统下、归类分组、逻辑递进

---

## 核心概念

```
          ┌─────────┐
          │  结论   │  ← 先说结论
          └────┬────┘
       ┌───────┼───────┐
       ▼       ▼       ▼
    ┌─────┐ ┌─────┐ ┌─────┐
    │论据1│ │论据2│ │论据3│  ← 再讲论据
    └──┬──┘ └──┬──┘ └──┬──┘
       │       │       │
       ▼       ▼       ▼
     细节    细节    细节   ← 最后细节
```

**四原则**：
1. **结论先行**：先说核心观点，再说支撑论据
2. **以上统下**：上层是下层的概括，下层是上层的支撑
3. **归类分组**：同组内容属于同一范畴
4. **逻辑递进**：按时间、结构、重要性等顺序排列

## 在技能文档中的应用

### 应用场景

| 场景 | 应用方式 |
|------|----------|
| README.md | 开头一句话定义（结论），再展开说明 |
| 章节组织 | 每章先写摘要（结论），再写详细内容 |
| 列表项 | 先写概括性标题，再展开说明 |

### 示例

```markdown
# cx-skill-test：技能快速测试方法

> 本技能提供结构化的快速测试流程，帮助验证技能的实际工作输出是否符合预期。

## 核心原则

- **快速**：单次验证，快速迭代
- **隔离**：测试阶段不安装到运行时环境
- **真实**：使用真实场景，不虚构用例
```

## 检查清单

- [ ] 每段开头是否有结论句？
- [ ] 上层观点是否能概括下层内容？
- [ ] 同组内容是否属于同一范畴？
- [ ] 内容排列是否有明确逻辑顺序？
