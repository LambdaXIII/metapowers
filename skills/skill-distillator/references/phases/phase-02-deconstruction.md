# 第二阶段：解构

> 拆解被蒸馏技能的能力结构和知识类型。

---

## 进入条件

- 前置输出：phase-01 的理解摘要（含元信息、黄金圈、冰山模型、5W1H、技能类型、源一致性报告、语言标签）
- 确认：phase-01 `阶段完成确认` 已通过

## 目标

将技能拆解为可重组的能力单元和知识单元，为后续编排做准备。

## 为什么需要这些知识文档

理解阶段建立了整体认知，但技能内部的能力结构和知识分布仍不清晰。需要：
- **功能分解**：自顶向下拆解，理清能力层级关系
- **知识类型分析**：识别不同类型的知识，便于后续按 Diátaxis 框架分类

## 如何加载知识文档

按顺序加载：
1. [frameworks/functional-decomposition.md](../frameworks/functional-decomposition.md) — 拆解能力结构
2. [frameworks/knowledge-types.md](../frameworks/knowledge-types.md) — 识别知识类型

加载方式：读取文件内容，提取分析方法，应用到被蒸馏技能。

## 操作步骤

### 步骤1：功能分解

- 识别核心能力：技能最本质的用途
- 拆解主要功能：为实现核心能力需要的功能模块
- 细化子功能：每个主要功能包含的具体操作

### 步骤2：知识类型分析

遍历技能文件，将知识点分类：
- **SKILL.md frontmatter**：从 `name`、`description`、`metadata` 提取概念知识与元认知知识
- **SKILL.md body**：主要来源，提取所有五种知识类型
- **references/**：概念知识、程序知识与规则知识的补充来源
- **README.md**（如有）：元认知知识与概念知识的重要来源（设计思想、决策理由）
- **templates/**（如有）：程序知识（模板使用步骤）和案例知识（模板本身作为示例）
- **examples/**（如有）：案例知识的主要来源
- **CHANGELOG.md**（如有）：元认知知识（版本演进理由）

知识分类：
- **概念知识**：术语定义、核心概念
- **程序知识**：操作步骤、流程
- **规则知识**：触发条件、约束
- **案例知识**：示例、场景
- **元认知知识**：设计思想、决策理由

### 步骤2.5：知识分布评估

统计各知识类型占比，评估知识分布特征：

- 统计五种知识类型的数量和占比
- 识别分布特征：
  - **程序知识主导**：程序知识 > 60% → 标记，影响编排和表达阶段策略
  - **设计理由不足**：元认知知识 < 10% → 标记，表达阶段需授权推断
- 将评估结果传递给后续阶段

### 步骤3：输出能力清单和知识图谱

- 能力清单：核心能力 → 主要功能 → 子功能
- 知识图谱：知识点按类型分类，标注来源文件
